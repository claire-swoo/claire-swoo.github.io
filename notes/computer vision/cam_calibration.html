Camera Calibration: matrices, QR decomposition, and Gram-Schmidt

Apr 2026 · Concept notes · Computer Vision
Core question.
In camera calibration, what do the main matrices mean, and why do decompositions such as QR or Gram-Schmidt appear when we want to recover intrinsic parameters?

1. Big picture

Camera calibration is about recovering the mapping from 3D world points to 2D image pixels. The point is not just to fit a matrix mechanically. The point is to separate three different things:

  • the camera internal geometry such as focal length and principal point,
  • the camera pose relative to the world,
  • and the final projection from 3D to image coordinates.

If those are mixed together, the matrix works numerically but tells you very little physically.

2. Core mapping: from 3D point to image pixel

In homogeneous coordinates, a 3D point X = [X, Y, Z, 1]^T is mapped to an image point x = [u, v, 1]^T by a projection matrix P:

x ~ P X

where ~ means equality up to scale. The standard form is

P = K [R | t]

This equation is the spine of camera calibration.

3. What each matrix means

3.1 Intrinsic matrix K

K describes how the camera itself converts coordinates in the camera frame into pixel coordinates. A common form is

K = [ [f_x, s, c_x], [0, f_y, c_y], [0, 0, 1] ]
  • f_x, f_y: focal length measured in pixel units along each image axis,
  • c_x, c_y: principal point, usually near the image center,
  • s: skew, often zero in modern cameras.

The focal length here is not simply “millimeters” unless you are working before pixel scaling. In the calibration matrix, it is usually expressed in pixel units.

3.2 Rotation matrix R

R tells you how the world coordinate axes are oriented relative to the camera. It is orthonormal, so its columns are mutually perpendicular unit vectors.

R^T R = I, det(R) = 1

3.3 Translation vector t

t tells you where the world origin is located relative to the camera frame, after the rotation convention is fixed.

3.4 Extrinsic matrix [R | t]

This combines rotation and translation, so it maps world coordinates into the camera coordinate frame.

4. Canonical camera and why it matters

The canonical camera strips away real camera details and keeps only the ideal projective mapping. The canonical projection matrix is often written as

P_c = [I | 0]

This means the camera is placed at the origin, aligned with the coordinate axes, with no separate intrinsic scaling yet.

It is useful because it shows what projection fundamentally is before adding calibration and pose. Real cameras are then obtained by applying extrinsics and intrinsics on top of this basic structure.

5. Simple projection example

Suppose a 3D point in the camera frame is (X_c, Y_c, Z_c). Under the ideal pinhole model, normalized image coordinates are

x_n = X_c / Z_c, y_n = Y_c / Z_c

Then the intrinsic matrix turns these into pixels:

u = f_x x_n + c_x, v = f_y y_n + c_y

Example: if f_x = f_y = 800, c_x = 320, c_y = 240, and a point in camera coordinates is (1, 2, 4), then

x_n = 1/4 = 0.25, y_n = 2/4 = 0.5
u = 800(0.25) + 320 = 520
v = 800(0.5) + 240 = 640

So the image point is approximately (520, 640).

6. Where QR decomposition enters

Once you estimate a camera matrix P, the left 3x3 block contains the mixed effect of intrinsics and rotation. Call that block M:

P = [M | p_4], with M = K R

The problem is that M mixes two different structures:

  • K should be upper triangular,
  • R should be orthonormal.

Decomposition is how we separate those two roles.

6.1 Why QR fits the structure

QR decomposition writes a matrix as

A = Q R

where Q is orthonormal and R is upper triangular. That looks close to what we want, but the order is reversed relative to M = K R.

This is why calibration texts often use RQ decomposition in practice, because it produces an upper-triangular matrix on the left and an orthonormal matrix on the right:

M = K R

So the clean conceptual statement is this:

If you want intrinsics × rotation, plain QR has the wrong order, so people usually use RQ, or transform the matrix so that QR can still be applied indirectly.

If a lecture or derivation mentions QR, the real point is not the label. The real point is separating an orthonormal factor from an upper-triangular factor.

7. Where Gram-Schmidt enters

Gram-Schmidt is the classical geometric procedure that turns a set of linearly independent vectors into an orthonormal basis. It is deeply connected to QR because applying Gram-Schmidt to the columns of a matrix constructs the orthonormal matrix in QR decomposition.

7.1 Basic idea

Suppose we start with vectors a_1, a_2. Gram-Schmidt does:

q_1 = a_1 / ||a_1||
u_2 = a_2 - (q_1^T a_2) q_1
q_2 = u_2 / ||u_2||

So the second vector is cleaned by subtracting the part that lies along the first. Repeating this process produces orthonormal directions.

7.2 Tiny numerical example

Let

a_1 = [1, 1]^T, a_2 = [1, 0]^T

First,

q_1 = (1 / sqrt(2)) [1, 1]^T

Project a_2 onto q_1:

q_1^T a_2 = 1 / sqrt(2)
(q_1^T a_2) q_1 = [1/2, 1/2]^T

Remove that component:

u_2 = [1, 0]^T - [1/2, 1/2]^T = [1/2, -1/2]^T

Normalize:

q_2 = (1 / sqrt(2)) [1, -1]^T

Now q_1 and q_2 are orthonormal.

8. Why this matters in calibration, not just algebra

The decomposition is not there because linear algebra books like factorization. It is there because the camera matrix contains different physical meanings that obey different structural constraints.

  • K must look like a calibration matrix, not an arbitrary matrix.
  • R must be a true rotation, not a distorted transform.

Decomposition is what forces the recovered parameters back into geometrically meaningful form.

9. Common confusion points

  • Focal length units: in K, focal length is usually in pixel units.
  • QR versus RQ: for calibration, the desired factor order is usually upper-triangular first, orthonormal second.
  • Homogeneous coordinates: equal up to scale means the same image point can be represented by many scalar multiples.
  • Projection matrix meaning: fitting P is only the start. Interpreting it requires factorization into physically meaningful parts.

10. Main takeaway

The camera calibration pipeline is easiest to understand when the roles are kept separate. The projection matrix P maps 3D to 2D, the intrinsic matrix K captures the camera's internal geometry, and [R|t] captures pose. QR, RQ, and Gram-Schmidt are not side tricks. They are the tools that let us separate these roles while preserving the geometry each factor is supposed to represent.
